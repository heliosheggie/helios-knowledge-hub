# ⚙️ Helios Operational Protocols

**Version:** 1.0
**Status:** Active
**Objective:** Ensure high-velocity, high-trust output through a structured pipeline.

## 🔄 The Value Pipeline (Research $\rightarrow$ Content $\rightarrow$ Community)

All Helios assets must follow the **Verified Flow**:

### Step 1: Intelligence Gathering (Research Agent)
- **Input:** Strategic goal or community knowledge gap.
- **Process:** Source data, validate technical specs, cross-reference UK market rates.
- **Output:** `RESEARCH_NOTE_[ID].md` containing raw, verified facts.

### Step 2: Asset Packaging (Content Agent)
- **Input:** `RESEARCH_NOTE_[ID].md`.
- **Process:** Translate technical data into homeowner-friendly formats (guides, tables, lists).
- **Output:** `CONTENT_DRAFT_[ID].md`.

### Step 3: Quality Assurance (The Loop)
- **Process:** The Research Agent must review the `CONTENT_DRAFT` to ensure no "meaning was lost in translation."
- **Sign-off:** Research Agent marks draft as `VERIFIED`.

### Step 4: Distribution & Feedback (Community Agent)
- **Input:** `VERIFIED CONTENT_DRAFT`.
- **Process:** Publish to hub, share in forums, engage users.
- **Output:** User feedback and new "Knowledge Gaps" $\rightarrow$ feeds back into Step 1.

## ⚖️ Decision Matrix

| Decision Level | Authority | Escalation Trigger |
| :--- | :--- | :--- |
| **Tactical** (How to write a post) | Agent Lead | None |
| **Strategic** (What topic to research) | CEO Agent | Alignment with Vision |
| **Critical** (Cost, Budget, Brand Shift) | The Board (Helios) | Any monetary cost or pivot |
